## What this is

A self-contained Flask app serving 888 research graphs from the ICU-EHR pipeline comparison project. The app has no database — all data lives in `graphs_manifest.json` and static PNGs under `static/graphs/`. It runs via Docker or a bare `./run.sh`.

## Running locally

```bash
# Docker (recommended):
docker compose up --build
# → http://localhost:8000

# Without Docker (auto-creates .venv/):
./run.sh
PORT=9000 ./run.sh

# Live reload during development:
FLASK_DEBUG=1 python app/server.py
```

After editing frontend files, restart with `docker compose restart` (no rebuild needed unless Python deps change).

## Architecture

**Data flow:** `graphs_manifest.json` → `/api/manifest` → `app.js` → cascading dropdowns → `renderPane()`

- `app/server.py` — three routes only: `/` (index.html), `/api/manifest` (JSON), `/graphs/<path>` (static PNGs/SVGs served from `static/graphs/`)
- `app/static/app.js` — all UI logic in vanilla JS (no build step). Key functions:
  - `initDropdowns()` — wires the 6-level cascading selects (dataset → pipeline → category → label → aggregation → graph), shared for main pane and both compare panes
  - `renderMainPane()` / `renderPane()` — display a graph record and its suggested cross-pipeline pairings
  - `toggleCompare()` — switches between single-pane and side-by-side layout
  - `visibleRecords()` — filters out `superseded` and `auto_named` records unless `SHOW_ALL` is true
- `graphs_manifest.json` — single source of truth for all dropdowns and suggestion pairs; generated by `build_manifest.py` on nibi (not in this repo)

## Manifest schema

Each record in `graphs_manifest.json["records"]` has: `id`, `pipeline` (MIMIC_Extract | EHR-Dataset-Processing | Comparison), `dataset` (mimic-iii | mimic-iv | eicu), `notebook`, `category`, `plot_type`, `label`, `aggregation`, `vital`, `scenario`, `filter`, `title`, `description`, `file`, `svg`, `thumb`, `auto_named`, `superseded`, `suggested_pairs`.

`auto_named: true` means the title was heuristically derived from cell source (marked with `*` in UI). `superseded: true` means hidden by default behind the "Show All" toggle.

## Key constraints

- **Graph generation runs on nibi (Compute Canada HPC), not on this machine.** The scripts `extract_notebook_graphs.py`, `render_marimo_mimic_iii.py`, and `build_manifest.py` are gitignored — they contain nibi-specific absolute paths. To add/update graphs, work on `l5.nibi.sharcnet`, then `git push` and `git pull` here.
- `static/graphs/` (~1 GB of PNGs) is committed to git — first clone is slow by design.
- The Dockerfile copies `static/graphs/` and `graphs_manifest.json` at build time; no volume mounts.

## Known issues

1. **Marimo mimic-iv/eICU not regenerated** — those 460 graphs come from legacy `*-analysis.ipynb` notebooks; regenerating requires Docker+GPU+DB environment on nibi.
2. **McNemar test results are unreliable** — the EHR predictions in `pipeline_comparison.ipynb` cell 20 use a dummy classifier. Accuracy/F1/AUC metrics are correct.
3. **625 of 888 graphs are auto-named** — improving names requires editing `CURATED_CATALOG` in `extract_notebook_graphs.py` on nibi and re-running extraction. Key is `(notebook_slug, output_index)` matching the `id` field (e.g. `experiment_statistics_0003` → slug=`experiment_statistics`, idx=3).
