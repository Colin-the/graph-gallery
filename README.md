# ICU-EHR Graph Gallery

Browser-based viewer for all graphs generated by the MIMIC_Extract and EHR-Dataset-Processing
pipelines, plus the cross-pipeline comparison notebook.

## Quick start (without Docker)

```bash
cd /home/ccampb47/work/graph-gallery
source /home/ccampb47/work/MIMIC_Extract/.venv_stats/bin/activate
pip install flask         # if not already installed in venv
python app/server.py      # serves on http://localhost:8000
```

## Building the graph library

Run these steps in order before starting the server (or after adding new notebooks):

### Step 1 — Extract embedded graphs from executed notebooks

```bash
source /home/ccampb47/work/MIMIC_Extract/.venv_stats/bin/activate
python extract_notebook_graphs.py          # ~500 images
python extract_notebook_graphs.py --check  # dry-run count only
```

Pulls every `image/png` base64 output from all executed `.ipynb` files in both pipelines and
the comparison notebook. Streams EHR branch notebooks via `git show` without checking out the
branch. Writes `static/graphs/extracted_manifest_fragment.json`.

### Step 2 — Render EHR Marimo notebook.py plots for MIMIC-III (SLURM)

```bash
sbatch render_marimo.sbatch
# Monitor:
squeue -u ccampb47 -j <jobid>
tail -f logs/render_marimo_<jobid>.out
```

Uses existing `Data/mimic-iii/` caches — no GPU or DB required. Renders ~70 plots
(length surface/heatmap, filter-impact bars, McNemar bars, centroid shift/density).
Writes `static/graphs/marimo_manifest_fragment.json`.

To render just one section quickly:
```bash
SECTION=impact sbatch render_marimo.sbatch
SECTION=centroid sbatch render_marimo.sbatch
```

### Step 3 — Build unified manifest

```bash
python build_manifest.py
```

Merges both fragments into `graphs_manifest.json`, computes cross-pipeline
suggested-pair links, and builds the dropdown option tree.

## Running with Docker (on a host that has Docker)

After completing the build steps above (so `static/graphs/` and `graphs_manifest.json` exist):

```bash
docker compose up --build
# Then open http://localhost:8000
```

Or without compose:
```bash
docker build -t ehr-gallery .
docker run -p 8000:8000 ehr-gallery
```

## Adding new graphs

1. If from a new executed notebook: add an entry to `NOTEBOOKS` in `extract_notebook_graphs.py`
   and re-run Step 1.
2. If from the Marimo `notebook.py`: add the plotting calls to `render_marimo_mimic_iii.py`
   and re-submit the SLURM job.
3. Always re-run `build_manifest.py` after any change.

## Storage / quota

Extracted PNGs are stored under `static/graphs/`. Typical sizes:
- ~500 notebook-extracted images ≈ 150–300 MB
- ~70 Marimo-rendered images ≈ 20–50 MB

If the home quota is tight, set `OUT_DIR` in `render_marimo_mimic_iii.py` and `STATIC_GRAPHS`
in `extract_notebook_graphs.py` to a path under `$SCRATCH`, then symlink:
```bash
ln -s $SCRATCH/graph-gallery/static/graphs /home/ccampb47/work/graph-gallery/static/graphs
```

## Notes

- MIMIC-IV and eICU EHR-Dataset-Processing graphs come from the legacy `*-analysis.ipynb`
  notebooks (438 pre-rendered PNGs on `origin/optimization_refactor`); they require
  Docker+GPU+DB to regenerate from the Marimo `notebook.py`.
- The `*` suffix on graph names in the dropdown means the title was auto-detected from
  cell source rather than hand-curated. Toggle **Show All** to also see superseded/exploratory
  notebooks (hidden by default).
- The EHR branch notebooks are read via `git show origin/optimization_refactor:<path>` so
  the working tree always stays on `master`.
